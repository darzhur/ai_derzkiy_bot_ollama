# Telegram бот с интеграцией ChatGPT и YandexGPT

Персональный дерзкий, ворчливый и саркастичный помощник на базе Telegram с поддержкой двух AI моделей:
- **ChatGPT** через ProxyAPI
- **YandexGPT** через Yandex Cloud ML SDK

## Требования

- Python 3.8+
- Токен Telegram бота (от @BotFather)
- ProxyAPI ключ (от https://proxyapi.ru)
- *(опционально)* Параметры Yandex Cloud для YandexGPT

## Установка
0. Установить Python
python3.11+

Установить Ollama
curl -fsSL https://ollama.com/install.sh | sh

Скачать модель
ollama pull llama3.2

Запуск на VPS
Минимальные требования для Ollama:
модель	RAM
llama3.2 3b	8 GB
mistral 7b	16 GB
llama3 8b	16-24 GB

Если VPS слабый: лучше llama3.2:3b

```bash

```

### 2. Создание виртуального окружения
```bash

source venv/bin/activate  # на macOS/Linux
```

```bash

```

### 4. Настройка переменных окружения
```bash

# Открываем и заполняем .env с вашими токенами
# TELEGRAM_BOT_TOKEN=your_token
# PROXYAPI_KEY=your_proxyapi_key
# YANDEX_FOLDER_ID=your_folder_id (опционально)
# YANDEX_AUTH_TOKEN=your_auth_token (опционально)
```

## Использование

### Запуск бота
```bash
python bot.py
```

Бот начнет слушать сообщения и отвечать на все текстовые запросы используя выбранную модель.

## Получение токенов и ключей

### Telegram токен
1. Откройте Telegram и найдите @BotFather
2. Отправьте команду `/newbot`
3. Следуйте инструкциям
4. Скопируйте полученный токен

### ProxyAPI ключ
1. Перейдите на https://proxyapi.ru
2. Авторизуйтесь или создайте аккаунт
3. Перейдите в раздел API ключей
4. Скопируйте ваш ключ авторизации

### YandexGPT параметры (опционально)
1. Создайте Folder в Yandex Cloud Console
2. Скопируйте Folder ID
3. Создайте IAM токен: `yc iam create-token`
4. Добавьте в .env файл

- `/model` - Выбрать модель (ChatGPT или YandexGPT)
- `/models` - Показать доступные модели
- `/help` - Справка по возможностям

## Выбор модели

По умолчанию используется **ChatGPT/ProxyAPI**. Для переключения на другую модель:

1. Отправьте команду `/model`
2. Выберите нужную модель из кнопок
3. Все последующие сообщения будут использовать выбранную модель

Выбор сохраняется для каждого пользователя во время сессии бота.

## Архитектура

### Основные функции

- `get_gpt_response()` - Отправляет сообщение в OpenAI и получает ответ
- `handle_text_message()` - Обработчик текстовых сообщений
- `handle_start()` - Обработчик команды /start
- `handle_help()` - Обработчик команды /help

### Поток данных

1. Пользователь отправляет текстовое сообщение
2. Бот получает сообщение
3. Бот отправляет сообщение в OpenAI с системным сообщением
4. OpenAI возвращает ответ
5. Бот отправляет ответ пользователю

## Обработка ошибок

Бот обрабатывает следующие ошибки:
- Ошибки аутентификации OpenAI
- Превышение лимита запросов
- Ошибки API
- Общие исключения

Все ошибки логируются, и пользователю отправляется информативное сообщение об ошибке.

## Переменные окружения

- `TELEGRAM_BOT_TOKEN` - Токен Telegram бота
- `PROXYAPI_KEY` - API ключ ProxyAPI для маршрутизации запросов к OpenAI

## Примечания

- История диалога не сохраняется (каждое сообщение независимо)
- Используется модель `gpt-4.1`
- Система логирует все действия бота
- Все запросы маршрутизируются через ProxyAPI (https://api.proxyapi.ru/openai/v1)
- Авторизация в ProxyAPI осуществляется через заголовок `Authorization: Bearer <KEY>`

## Решение проблем

### "TELEGRAM_BOT_TOKEN не установлен"
Убедитесь, что вы:
- Создали файл `.env`
- Заполнили `TELEGRAM_BOT_TOKEN`
- Загрузили переменные перед запуском (можно использовать `python-dotenv`)

### "PROXYAPI_KEY не установлен"
Убедитесь, что вы:
- Зарегистрировались на https://proxyapi.ru
- Скопировали ваш API ключ в переменную `PROXYAPI_KEY`
- Ключ корректно передается в заголовке авторизации

### "Ошибка подключения к ProxyAPI"
Проверьте:
- Интернет соединение
- Корректность ключа ProxyAPI
- Статус сервиса на https://proxyapi.ru
- Что используется правильный endpoint: `https://api.proxyapi.ru/openai`
